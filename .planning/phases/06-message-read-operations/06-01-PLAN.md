<objective>
Implement two new read-only message commands: `tg message get <chat> <ids>` for fetching specific messages by ID with batch support and not-found tracking, and `tg message pinned <chat>` for listing pinned messages with pagination. Extract `buildEntityMap` from replies.ts to a shared utility for reuse across all three commands.

Purpose: Fulfills READ-08 and READ-09 requirements. These commands enable agents to verify message existence and discover pinned messages -- both common workflows for Telegram automation.

Output: Two working CLI subcommands, shared entity-map utility, human-readable format support, and comprehensive unit tests.
</objective>

function buildEntityMap(result: any): Map<string, any> {
  const map = new Map<string, any>();
  for (const u of result.users ?? []) {
    map.set(u.id.toString(), u);
  }
  for (const c of result.chats ?? []) {
    map.set(c.id.toString(), c);
  }
  return map;
}
