  <action>
    **1. Create src/commands/media/download.ts:**

    Export `mediaDownloadAction` handler following established patterns (see message/send.ts, message/search.ts):

    ```
    async function mediaDownloadAction(this: Command): Promise<void>
    ```

    - Parse `this.args[0]` as chat, `this.args[1]` as comma-separated message IDs.
    - Parse IDs: split by comma, parseInt each, reject NaN with error.
    - Get opts: `output` (-o flag), `quiet`, `profile`, `config`.
    - Use `store.withLock` -> `withClient` pattern (same as all other commands).
    - Resolve entity via `resolveEntity(client, chat)`.

    For each message ID:
    - `const messages = await client.getMessages(entity, { ids: [msgId] })` — fetch the message.
    - `const msg = messages[0]` — if not found or no media, throw `TgError('Message ${msgId} has no downloadable media', 'NO_MEDIA')`.
    - Call `extractMediaInfo(msg.media)` for metadata and `detectMedia(msg.media)` for mediaType.
    - Determine filename: `mediaInfo?.filename ?? generateFilename(mediaType ?? 'file', msgId, mediaInfo?.mimeType)`.
    - Determine target path:
      - Single download + `-o` is file path: `resolve(opts.output)`
      - Batch download + `-o` is directory: `resolve(opts.output, filename)`
      - No `-o`: `resolve(process.cwd(), filename)`
    - Call `await client.downloadMedia(msg, { outputFile: targetPath, progressCallback })`.
    - Progress callback: throttle to max once per second. Use `logStatus` with `quiet` flag. Format: `Downloading ${filename}: ${pct}% (${formatBytes(downloaded)}/${formatBytes(total)})`. Convert BigInteger values with `.toJSNumber()`.
    - Build DownloadResult: `{ path: targetPath, filename, size: mediaInfo?.fileSize ?? 0, mediaType: mediaType ?? 'unknown', messageId: msgId }`.

    Single download: `outputSuccess(result)`.
    Batch download: `outputSuccess({ files: results, downloaded: results.length })`.

    Wrap in try/catch with `formatError`.

  <behavior>
    - Single file send: calls client.sendFile with file path, returns serialized MessageItem
    - Album send (multiple files): calls client.sendFile with array, re-fetches album messages, returns { messages: MessageItem[], sent: N }
    - Caption flag: passes caption to sendFile
    - Reply flag: passes replyTo to sendFile
    - Voice note detection: .ogg/.opus files set voiceNote: true
    - File not found: returns FILE_NOT_FOUND error before sending
    - Album > 10 files: returns ALBUM_TOO_LARGE error
    - Progress output on stderr (suppressed with --quiet)
    - Auto type detection: .jpg sends as photo, .mp4 as video, .pdf as document (no --type flag)
  </behavior>
  <action>
    **1. Create src/commands/media/send.ts:**

    Export `mediaSendAction` handler:

    ```
    async function mediaSendAction(this: Command): Promise<void>
    ```

    - Parse `this.args[0]` as chat, remaining args as file paths (variadic).
    - Get opts: `caption`, `replyTo` (parse to int), `quiet`, `profile`, `config`.
    - Validate file paths exist using `fs.access()` for each — throw `TgError('File not found: ${fp}', 'FILE_NOT_FOUND')` on failure.
    - Validate album size: if files.length > 10, throw `TgError('Albums support a maximum of 10 files', 'ALBUM_TOO_LARGE')`.
    - Use `store.withLock` -> `withClient` pattern.
    - Resolve entity via `resolveEntity(client, chat)`.

    Determine send params:
    ```typescript
    const isAlbum = files.length > 1;
    const sendParams: any = {
      file: isAlbum ? files.map(f => resolve(f)) : resolve(files[0]),
      caption: opts.caption ?? '',
      replyTo: opts.replyTo,
      progressCallback: (progress: number) => {
        if (!opts.quiet) {
          logStatus(`Uploading: ${Math.round(progress * 100)}%`);
        }
      },
    };
    ```

    Voice note detection: check first file extension with `detectFileType`. If result is `'voice'`, set `sendParams.voiceNote = true`.

    Force document for non-photo/video/voice: for single file uploads, if `detectFileType(ext)` returns `'document'`, set `sendParams.forceDocument = true` to prevent gramjs from attempting image detection on non-image files.

    Send: `const result = await client.sendFile(entity, sendParams);`

    Handle response:
    - Single file: `result` is an Api.Message. Serialize with `serializeMessage(result)`. `outputSuccess(serialized)`.
    - Album: `_sendAlbum` returns a single message. To get all album messages, use the returned message's ID to fetch nearby messages. Strategy: the album messages are sequential, so fetch messages with IDs from `(result.id - files.length + 1)` to `result.id`:
      ```typescript
      const ids = Array.from({ length: files.length }, (_, i) => result.id - files.length + 1 + i);
      const albumMsgs = await client.getMessages(entity, { ids });
      const serialized = albumMsgs.filter(Boolean).map(m => serializeMessage(m));
      outputSuccess({ messages: serialized, sent: serialized.length });
      ```
      If the sequential ID assumption fails (some messages are null), fall back to just returning the single message.

    Wrap in try/catch with `formatError`.
