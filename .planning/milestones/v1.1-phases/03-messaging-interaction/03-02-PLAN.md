  <action>
1. Create `src/lib/format.ts`:
   - Import picocolors as `pc`
   - Import types: MessageItem, ChatListItem, ChatInfo, MemberItem, SearchResultItem
   - `formatMessages(messages: MessageItem[]): string` -- conversational format per CONTEXT.md:
     ```
     [2026-03-11 12:30] Alice: Hello world
     [2026-03-11 12:31] Bob (reply to 42): Thanks!
     [2026-03-11 12:32] Alice: [photo] Check this out
     ```
     Use `pc.dim()` for timestamp, `pc.bold()` for sender name, `pc.dim()` for reply annotation, `pc.yellow()` for media tag. Format date from ISO string: `new Date(m.date)` -> `YYYY-MM-DD HH:MM` in local time.
   - `formatChatList(chats: ChatListItem[]): string` -- table format:
     ```
     [channel]    My Channel @mychan
     [supergroup] Dev Group @devs (5 unread)
     [user]       Alice @alice
     ```
     Use `pc.dim()` for type tag, `pc.bold()` for title, `pc.dim()` for @username, `pc.yellow()` for unread count. Left-pad type to consistent width.
   - `formatChatInfo(info: ChatInfo): string` -- key-value pairs:
     ```
     Title:       My Channel
     Type:        channel
     Username:    @mychan
     Description: A test channel
     Members:     1,234
     ```
     Use `pc.bold()` for labels, skip null values.
   - `formatMembers(members: MemberItem[]): string` -- one line per member:
     ```
     Alice @alice
     Bob @bob [bot]
     ```
     Use `pc.bold()` for name, `pc.dim()` for @username, `pc.cyan()` for [bot] tag.
   - `formatSearchResults(results: SearchResultItem[]): string` -- group by chatTitle:
     ```
     --- Dev Group ---
     [2026-03-11 12:30] Alice: found this
     --- General Chat ---
     [2026-03-11 12:35] Bob: also here
     ```
     Use `pc.bold()` for chat title headers, then delegate each message to single-message format.
   - `formatGeneric(data: unknown): string` -- fallback: `JSON.stringify(data, null, 2)` for unknown data shapes. Used for auth status, session export/import, join/leave confirmations, etc.
   - Export a `formatData(data: unknown): string` dispatcher function that auto-detects data shape:
     - Has `.messages` array where items have `.text` and `.date` -> formatMessages
     - Has `.chats` array where items have `.type` and `.title` -> formatChatList
     - Has `.title` and `.type` and `.memberCount` (ChatInfo shape) -> formatChatInfo
     - Has `.members` array where items have `.isBot` -> formatMembers
     - Has `.messages` array where items have `.chatTitle` (SearchResultItem) -> formatSearchResults
     - Default: formatGeneric

   - Export `setOutputMode(human: boolean): void` -- sets `_humanMode = human`
   - Export `getOutputMode(): boolean` -- returns `_humanMode`
   - Modify `outputSuccess<T>(data: T): void`:
     - If `_humanMode`: import and call `formatData(data)` from format.ts, write result to stdout via `process.stdout.write(result + '\n')`
     - Else: existing JSON envelope behavior (unchanged)
   - Modify `outputError(error: string, code?: string): void`:

     - Else: existing JSON envelope to stdout (unchanged)

   - Add a `hook('preAction')` on the root program:
     ```typescript
     program.hook('preAction', (thisCommand) => {
       const opts = thisCommand.optsWithGlobals();

     });
     ```
   - This means NO action handlers need to check the flag themselves -- outputSuccess auto-dispatches.
