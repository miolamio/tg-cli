1. Create `src/commands/session/export.ts`:
   - Export `async function exportAction(this: Command)`
   - Get global opts via `this.optsWithGlobals()` -- extract profile, quiet, json/human
   - Create config and SessionStore
   - Load session for profile
   - If empty: outputError("No session found for profile '${profile}'", "NO_SESSION"), return
   - If human mode or default (not --json): write raw session string to stdout via `process.stdout.write(sessionString + '\n')`
   - If --json explicitly requested OR json is default: look up profile data from config, output `{ session: sessionString, phone: profileData?.phone, created: profileData?.created }`

   Note: The raw string output mode is important because session strings are meant to be piped. The JSON envelope mode adds metadata. Since JSON is the app default, check if user explicitly passed --json vs it being the default. Implementation detail: use Commander's `getOptionValueSource('json')` to detect if --json was explicitly passed. If the source is 'default', output raw string (export's special behavior for piping). If source is 'cli', output JSON envelope.

2. Create `src/commands/session/import.ts`:
   - Export `async function importAction(sessionString: string | undefined, this: Command)`
   - Commander config: `.argument('[session]', 'Session string to import')`
   - Get global opts -- extract profile, quiet
   - If sessionString argument provided: use it directly
   - If no argument: check if stdin is a TTY. If not TTY (piped), read from stdin until EOF. If TTY: outputError("Provide session string as argument or pipe via stdin", "NO_INPUT"), return
   - Trim the session string, validate it's non-empty
   - Save to SessionStore: `store.save(profile, sessionString)`
   - Update config profile: `config.set(\`profiles.${profile}\`, { session: sessionString, created: new Date().toISOString() })`
   - Output success: `{ imported: true, profile }`

   - Shebang: `#!/usr/bin/env node` (tsup will also add this via banner, but having it in source doesn't hurt)

   - Read package version AND gramjs version:
     ```
     import { readFileSync } from 'node:fs'
     import { fileURLToPath } from 'node:url'
     import { dirname, join } from 'node:path'

     const __filename = fileURLToPath(import.meta.url)
     const __dirname = dirname(__filename)

     let pkgVersion = '0.0.0'
     let gramjsVersion = 'unknown'
     try {
       const pkg = JSON.parse(readFileSync(join(__dirname, '..', '..', 'package.json'), 'utf-8'))
       pkgVersion = pkg.version
     } catch {}
     try {
       const gramjsPkg = JSON.parse(readFileSync(join(__dirname, '..', '..', 'node_modules', 'telegram', 'package.json'), 'utf-8'))
       gramjsVersion = gramjsPkg.version
     } catch {}
     const versionString = `${pkgVersion} (gramjs ${gramjsVersion})`
     ```
     Note: For the built binary (dist/bin/tg.js), __dirname will be dist/bin/, so `../..` reaches the project root where both package.json and node_modules live. If the gramjs read fails (e.g., in a bundled install scenario), it gracefully falls back to "unknown".
   - Create root program:
     ```
     const program = new Command()
       .name('tg')
       .description('Telegram CLI - Agent-first Telegram client')
       .version(versionString, '-V, --version')
     ```
   - Add global options:
     - `--json` (default true -- JSON is default output)
     - `--human` (human-readable output)
     - `-v, --verbose` (extra info)
     - `-q, --quiet` (suppress stderr)
     - `--profile <name>` (named profile, default: "default")
     - `--config <path>` (config file path)
   - Enable positional options: `program.enablePositionalOptions()`
   - Configure help command grouping using Commander.js v14 API. Before adding each command, set its help group so `tg --help` displays commands organized by category per the locked user decision:
     ```
     const authCmd = createAuthCommand()
     authCmd.helpGroup('Auth')
     program.addCommand(authCmd)

     const sessionCmd = createSessionCommand()
     sessionCmd.helpGroup('Session')
     program.addCommand(sessionCmd)
     ```
     This produces help output that groups commands under "Auth:" and "Session:" headings instead of a flat list. Future phases will add groups for "Chat", "Message", and "Media" as those command groups are implemented.
   - Add aliases: These are Phase 2+ features, so don't wire aliases yet. Just note in a comment: `// Aliases (Phase 2+): tg ls -> chat list, tg s -> message search`
