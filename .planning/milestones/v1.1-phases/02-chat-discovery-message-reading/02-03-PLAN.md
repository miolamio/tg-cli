  <files>src/commands/message/history.ts, src/commands/message/search.ts, tests/unit/message-history.test.ts, tests/unit/message-search.test.ts</files>
  <behavior>
    - messageHistoryAction("chatname"): Resolves entity, calls client.getMessages(entity, { limit }), serializes each via serializeMessage, outputs { messages: MessageItem[], total: number }
    - messageHistoryAction with --limit 10 --offset 5: Uses offsetId or skip-based pagination to return correct page
    - messageHistoryAction with --since "2026-03-01": Filters messages to only those after the given date. Uses search API with empty string and minDate for server-side filtering, or post-filters if needed.
    - messageHistoryAction with --until "2026-03-10": Uses offsetDate as Unix timestamp to get messages before that date
    - messageHistoryAction with --since and --until: Applies both bounds
    - messageSearchAction with --chat "chatname" -q "keyword": Resolves entity, calls client.getMessages(entity, { search: "keyword", limit }), serializes via serializeMessage, outputs { messages: MessageItem[], total: number }
    - messageSearchAction with -q "keyword" (no --chat): Calls client.getMessages(undefined, { search: "keyword", limit }) for global search, serializes via serializeSearchResult with chatId/chatTitle from each message's peerId, outputs { messages: SearchResultItem[], total: number }
    - messageSearchAction without -q: Outputs error requiring --query flag
    - Messages sorted newest first (chronological descending) by default
  </behavior>
  <action>
    1. Create src/commands/message/history.ts:
       - Export `messageHistoryAction(this: Command, chatInput: string): Promise<void>`
       - Options type: `GlobalOptions & { limit: string; offset: string; since?: string; until?: string }`
       - Parse limit/offset to numbers
       - Resolve entity via resolveEntity(client, chatInput)
       - Build getMessages params:
         - `{ limit: parsedLimit }`
         - If --offset provided: use `offsetId` from calculating skip (fetch offset+limit messages, slice). Alternatively, if offset is a message ID, use it directly. For simplicity, use `addOffset: parsedOffset` which gramjs supports as a skip parameter.
         - If --until provided: parse ISO date string to Unix timestamp via `Math.floor(new Date(until).getTime() / 1000)`, set as `offsetDate`
         - If --since provided: Two strategies:
           a. If NO search query: use `search: ""` with the minDate param passed to the raw invoke API. gramjs `getMessages` passes `search` to `_MessagesIter` which uses `messages.Search` API supporting minDate.
           b. Alternatively: post-filter by checking `msg.date >= sinceTimestamp` and stop iteration when messages are before since date (since they're newest-first).
           Use strategy (b) for reliability: fetch messages, filter out those before --since. Stop if all messages in a batch are before --since.
       - Serialize each message via serializeMessage
       - Output: `outputSuccess({ messages: serialized, total: messages.total ?? 0 })`

    2. Create src/commands/message/search.ts:
       - Export `messageSearchAction(this: Command): Promise<void>`
       - Options type: `GlobalOptions & { chat?: string; query: string; limit: string; offset: string }`
       - Validate --query is provided; if not, `outputError('--query (-q) is required for search. Use `tg message history` to browse without a query.', 'MISSING_QUERY')` and return
       - Parse limit/offset
       - If --chat provided (per-chat search, READ-03):
         - Resolve entity via resolveEntity(client, chatInput)
         - Call `client.getMessages(entity, { search: query, limit: parsedLimit, addOffset: parsedOffset })`
         - Serialize each via serializeMessage
         - Output: `outputSuccess({ messages: serialized, total: messages.total ?? 0 })`
       - If --chat NOT provided (global search, READ-04):
         - Call `client.getMessages(undefined, { search: query, limit: parsedLimit })`
         - For each message, resolve chatId and chatTitle from msg.peerId:
           - Use `bigIntToString(msg.peerId?.channelId || msg.peerId?.chatId || msg.peerId?.userId)` for chatId
           - Try to get chatTitle from `msg.chat?.title` or `msg._chat?.title` if available; fall back to chatId
         - Serialize each via serializeSearchResult
         - Output: `outputSuccess({ messages: serialized, total: messages.total ?? 0 })`

         - Test: pagination with --limit and --offset
